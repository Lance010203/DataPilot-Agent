# DataPilot Agent｜数据处理与分析工作流助手

> 一个面向业务数据分析场景的轻量级 **AI Agent / Workflow Demo**：上传 CSV 或 Excel 后，自动完成字段识别、数据质量检查、业务场景推断、数据清洗建议、分析方向建议、SQL 模板和 Markdown 报告生成。

## v0.3 专业版更新

v0.3 将项目从基础数据处理 Demo 进一步升级为轻量级商业分析工作流助手：

1. 修复页面模块编号，正式流程从 `1. 数据上传` 开始；
2. 升级业务场景判断逻辑，引入字段组合、类型和值特征、置信度、匹配依据、备选场景和人工确认；
3. 增加关键业务指标模块，只计算字段存在且口径合理的指标；
4. 重做专业可视化模块，使用深蓝/灰蓝的简洁咨询风格，控制在 4–6 张图；
5. 每张图自动生成与场景和图表类型相关的业务 insight；
6. 分析建议升级为业务问题、优先方向、数据缺口、风险和交付物结构；
7. Markdown 报告升级为结论先行的初步商业分析报告；
8. 同时提供 SQL 与 Python / Pandas 分析模板。

**项目定位：**

“DataPilot Agent 不是简单的数据清洗脚本，而是一个面向业务数据分析前期流程的轻量级 Agent Workflow。它尝试模拟商业分析师拿到一份陌生数据后的工作路径：理解字段、判断业务场景、检查数据质量、提取关键指标、生成可视化、提出分析方向，并输出初步报告。”

## 项目背景

业务分析的第一步通常不是建模，而是反复确认字段、检查数据质量、理解业务口径并整理分析思路。这些步骤高度重复，也很适合被拆解成可解释的 Agent Workflow。

DataPilot Agent 使用 **Python + Pandas + Streamlit** 实现一条轻量级自动分析流程。第一版默认不依赖付费 API，通过规则逻辑和 Prompt Engineering 模板模拟 AI Agent 的“理解、检查、建议、生成”过程，优先保证项目能运行、能展示、能讲清楚。

## 核心功能

- 上传并读取 UTF-8 / GBK CSV 和 XLSX 文件；
- 支持一键加载内置订单样例，快速体验完整流程；
- 展示数据样例、行列数、字段名与字段类型；
- 统计每列缺失数量、缺失率和完全重复行；
- 输出数值字段描述性统计和类别字段摘要；
- 使用 IQR 方法识别潜在异常值；
- 根据字段关键词完成业务场景推断；
- 自动生成数据清洗建议和后续分析建议；
- 自动生成缺失值、数值分布、类别 Top10 和时间趋势图；
- 支持用户勾选后执行基础清洗，并下载 `cleaned_data.csv`；
- 结合实际字段生成 GMV、用户分层、履约、增长和风控分析建议；
- 根据实际字段生成 SQL 分析模板；
- 生成可展示、可下载的 Markdown 初步分析报告；
- 提供数据分析 Agent 和清洗建议 Agent 的 Prompt 模板。

## Workflow

```text

```

## 项目功能模块

Streamlit 页面按业务分析前期流程拆分为 11 个模块：

1. 数据上传；
2. 数据预览；
3. 数据概览；
4. 数据质量检查；
5. 业务场景判断；
6. 关键业务指标；
7. 专业可视化分析；
8. Human-in-the-loop 清洗与下载；
9. 商业分析建议；
10. SQL / Python 分析模板；
11. Markdown 报告生成。

## 专业场景判断

v0.3 不再根据单个字段强行分类，而是综合：

- 字段名称关键词；
- 订单、用户、商品、金额、时间等字段组合；
- 数值、类别和时间字段类型；
- 支付状态、风险标签、用户事件等字段值特征；
- 场景最低组合信号和置信度阈值。

当前支持订单/交易、配送/履约、营销/增长、风控/信用/预测、财务/经营、宏观经济/金融市场、用户行为/产品分析和通用业务数据。

页面会展示主场景、置信度、判断依据和备选场景。置信度低于 65% 时提示人工确认，用户也可以手动覆盖场景。后续 KPI、图表、建议和报告均基于最终确认场景。

## 关键业务指标

系统会根据确认后的业务场景和实际字段计算可用指标，例如：

- 订单：订单量、GMV、客单价、支付成功率、用户数；
- 配送：平均配送时长、超时率、平均距离；
- 营销：曝光、点击、CTR、转化、CVR、花费、ROI；
- 财务：收入、成本、毛利、毛利率、预算完成率；
- 宏观金融：时间范围、最新值、历史均值；
- 产品行为：活跃用户、事件量、会话时长。

字段不足时不会报错，而是在“暂不可计算的指标”中说明缺失字段。

## 专业可视化

系统根据数据字段自动选择可展示的图表：

- **缺失值可视化**：按字段展示缺失值数量；
- **数值字段分布**：对前几个数值字段进行分箱并展示基础分布；
- **类别字段 Top10**：优先选择非 ID 类别字段，展示最高频的 10 个值；
- **时间趋势**：识别日期或时间字段，按日聚合记录数量；
- **友好降级**：没有适合字段时提示原因，不中断整体 Workflow。

v0.3 使用 Matplotlib 统一白底、深蓝、灰蓝和弱网格风格，并根据字段组合选择：

- 时间趋势折线图；
- Top N 横向贡献排名；
- 数值分布直方图；
- 两个数值字段的散点关系；
- 适量数值字段的相关性热力图；
- 仅在存在缺失值时展示的缺失值排名。

页面最多展示 6 张图，每张图下方提供一句具体业务 insight，说明该图可以回答什么问题。

## Human-in-the-loop 清洗

系统不会默认修改用户数据。用户需要主动勾选以下操作：

- 删除完全重复行；
- 将字段名标准化为小写下划线；
- 自动识别并转换日期或时间字段。

页面会同时展示清洗前后行列数、已执行操作和高缺失字段提示。高缺失字段只提示，不会自动删除。清洗结果可以下载为 `cleaned_data.csv`，原始 DataFrame 始终保留。

这个流程体现了 Human-in-the-loop 思路：Agent 负责识别问题和提供操作入口，关键数据修改由用户确认。

## 商业分析建议

建议模块会结合业务场景和真实字段名生成可执行方向：

- **订单/交易**：GMV、订单量、客单价、用户分层、城市/品类表现、支付转化率；
- **配送/履约**：平均配送时长、P50/P90、超时率、距离分组、高风险订单；
- **营销/增长**：CTR、CVR、ROI、渠道对比、转化漏斗、获客成本；
- **风控/预测**：标签分布、样本不平衡、特征相关性、模型指标、高风险样本；
- **通用业务**：描述性统计、分组对比、趋势分析和异常样本排查。

例如数据中存在 `price`、`quantity`、`user_id` 和 `payment_status` 时，系统会直接引用这些字段说明 GMV 口径、用户分层和支付成功率分析方式。

```bash
python3 -m venv .venv

pip install -r requirements.txt
```


```bash
streamlit run app.py
```

## 项目亮点

1. **完整的 Agent Workflow**

   将数据读取、数据理解、质量检查、业务推断、建议生成和报告输出拆分为独立模块，体现 AI Agent 应用的任务编排思路。

2. **结果可解释**

   场景推断和清洗建议基于透明规则，能够说明触发关键词、缺失率和异常值数量，便于业务人员复核。

3. **不依赖付费 API**

   默认使用本地规则即可运行，降低 Demo 体验门槛；`prompts/` 中预留了接入大模型时可使用的系统提示词。

4. **面向真实业务分析流程**

   不只展示图表，而是覆盖字段识别、数据质量检查、SQL 分析模板和报告生成，更接近数据产品与分析 Workflow。

5. **模块清晰，便于扩展**

   各步骤均为独立 Python 函数，后续可接入 OpenAI 或本地大模型、数据库、可视化和任务历史管理。
